 5. Um compilador é um programa que transforma instruções simbólicas em 
    código de máquina executável.
 
 6. Um código fonte é o programa escrito por um programador seja em 
    linguagem de primeira, segunda ou terceira geração.
 
 7. Código de máquina são instruções em linguagem binária as quais são 
    executáveis pelo processador.
 
 8. Linguagem de Primeira Geração refere-se aos primórdios da programação, 
    nos quais os programadores inseriam instruções e dados diretamente em 
    código binário.
 
 9. Linguagem de Segunda Geração foi um salto enorme para a programação, uma 
    vez que instruções puramente binárias foram substituídas por instruções 
    em texto comum, simbólicas por não poderem ser executadas diretamente. 
    Dessa forma, precisavam passar por um compilador a fim de serem 
    transformadas em código de máquina executável.
 
 10. Linguagem de Terceira Geração são abstrações maiores ainda que as de 
    segunda geração, como a linguagem C e a linguagem Python. Nessa geração, 
    existem compiladores, linkers e interpretadores. Além disso, geralmente 
    as linguagens dessa geração possuem objetivos específicos, como 
    JavaScript para renderização no frontend e Rust para segurança de 
    memória.
 
 11. Um interpretador é um programa que executa código fonte sem transformar 
    em código de máquina, ou seja, sem compilar. Nesse sentido, é mais lento 
    que código compilado, mas é compatível com multiplataforma.
